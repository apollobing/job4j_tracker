Анализ работы программы tracker.

Анализ производился при помощи инструментов: jps, jmap, jstat и jconsole.

После запуска программы использовался jps для получения pid.
Были получены данные ДО добавления заявок в память.
Выполнена команда jmap для подтверждения отсутствия объектов Item в памяти.
Выполнена команда jstat для отображения состояния памяти ДО добавления заявок.
Запущен jconsole, который отобразил состояние памяти в виде графика.

После в память были добавлены 1000 объектов Item.
Выполнена команда jmap для подтверждения наличия 1000 объектов Item в памяти.
Выполнена команда jstat, которая показала рост используемой памяти.
jconsole позволил на графике наблюдать рост используемой памяти.

Затем были удалены 1000 объектов Item.
Выполнена команда jmap для подтверждения отсутствия объектов Item в памяти.
Выполнена команда jstat, показавшая изменения в памяти в сравнении с прошлым запуском.
jconsole позволил на графике наблюдать снижение используемой памяти.

Вывод: при добавлении объектов инструменты: jps, jmap, jstat и jconsole,
позволяют видеть динамику роста используемой памяти, а при удалении
наблюдать снижение используемой памяти.
